Overview

Description

  • Name: CMAcquisitionListScript_v1.2.py.
  • Date Created/Modified: 15-05-2025.
  • Version: 1.2.
  • Software Version(s):
    • Compound Discoverer 3.3 SP3 (CD 3.3.3) and 3.4 (CD 3.4).
    • Chromeleon 7.3.2 (CM 7.3.2).
    • Orbitrap Exploris GC Method Editor 4.1.3xx (OEGC ME 4.1.3) and above.
    • May be suitable for other versions of the above referenced software.

Purpose

  • Compound Discoverer Scripting Node Python script used to export data from Compound Discoverer as an Acquisition List.
    • Can be imported into a Chromeleon CDS Processing Method using the Import Compound Data feature.
    • Can be imported into Orbitrap Exploris GC Method Editor as a SIM Scan list.
    • For GC EI Workflows.
    • File Format: Exploris Orbitrap LCMS and GCMS Family (csv).
  • No Script Registration required.

General Mechanism

  • Exports data (tables) listed in the node parameter Requested Tables and Columns.
    • Data are exported as tab-separated text files.
    • Requires the following tables (in order): GC EI Compounds, GC EI Compounds per File, and Features per File.
    • The order of tables is important for the script.
    • Incorrect order and/or typographical errors in the names of tables will result in Execution Failure error.
  • Writes a node_args.json file:
    • Used by the script to receive information from the application.
    • Contains essential information about the exported data, including location of the text files and the row IDs and columns for the exported tables.
  • Does not produce a response (node_response.json) file – no results are imported/integrated back into Compound Discoverer result files.
  • Produces csv files, Acquisition List and SIM Scan List, in the format Exploris Orbitrap LCMS and GCMS Family.
  • Generates lists of compounds each with up to its top three (3) largest (by area counts) m/z ions.
  • The files Acquisition List.csv and SIM Scan List.csv are saved in the local user’s Downloads folder (C:\Users\[user.name\]\Downloads).

Usage

Adding Scripting Node to a Post-Processing Workflow

Adjusting Scripting Node Parameters

  • Path to Executable: enter the path to run the script from the command line:
    • Example for Python: c:\[Pythonversion]\Python.exe.
  • Command Line Arguments: Specifies the command line arguments passed to the executable.
    • Include the argument between quotations marks – Use Ctrl+Shift+C (Copy as path) Windows 11 command.
  • Requested Tables and Columns: Specifies the requested data tables and columns used by the executable.
    • Tables are separated by a semi-colon (spaces after semi-colons are supported).
  • Use R-Friendly Columns: set to True (recommended). Removes special characters from column headings and replaces them with alphanumeric values.
  • Archive Datafiles: set to True (recommended). Creates an archive of the text and JSON files (useful for debugging scripts).

Conventions

Important Considerations – Selected (Checked) Entries

  • The script checks whether entries have been Checked (marked with a Check mark; selected) in the GC EI Compounds table as well as all related tables (GC EI Compounds per File and Features per File) in the Compound Discoverer Result File.
    • If so, the script will generate Acquisition Lists file of the Checked entries only.
      • If this outcome is not desired, as a general good practice and to avoid confusion, be sure to uncheck all entries in all tables.
    • If not (i.e., entries have been partially marked as Checked or not at all), the script will generate Acquisition Lists of all entries as defined in the GC EI Compounds table.
  • Follows an “all or nothing” approach.
  • These conventions should allow the user to be very specific as to which compound and/or m/z ion entries to export.
  • For a less specific (and more general) approach, the user can simply export all entries from Compound Discoverer and deselect the undesired entries during the import process (i.e., while using the Chromeleon Import Compound Data feature).

Important Considerations – Base Compound Value

  • Regardless of whether entries are marked as Checked, the script only exports entries that are True Base Compounds.
  • For an entry to be exported, its Base Compound value must be set to True.
    • Column can be found in GC EI Compounds per File table.
  • This is consistent with Compound Discoverer’s designation of Base Compound – the basis (non-gap compound with the largest TIC) used to create the group at a retention time.
    • Avoids exporting incorrect (or non-ideal) m/z ion entries.
  • If an entry’s Base Compound value is not set to True, the entry will not be exported.
    • This includes any associated GC EI Compounds or Features per File entries.
    • The script will, instead, generate Acquisition Lists of all entries as defined in the GC EI Compounds table.

Important Considerations – Sorting of m/z Ions

  • By default, the script will generate Acquisition Lists of compounds with each entry having up to three (3) m/z ions.
  • The three m/z ions are those that have the three largest Area counts.
    • Data can be reviewed in the Features per File table.
  • The user can override this behavior and select up to 3 different m/z ions by selecting (Checking) each GC EI Compounds entry along with its corresponding related tables entries.
    • This must be performed for every desired compound entry in all tables.
    • Follows an “all or nothing” approach.
  • The m/z ions, in order of decreasing area counts, will then be assigned to Chromeleon’s Processing Method Component Table as MS Quantitation Peak, MS Confirming Peak 1, and MS Confirming Peak 2, respectively.
